2. Файл S.java некорректно отформатирован: несколько методов написаны в одну строку, съезжают закрывающиеся скобки.
3. В исходноназванном методе int funct в классе S используются блоки if, которые можно заменить на конструкцию switch-case.
4. Нейминг класса S и методов в нём является неочевидным для пользователя (программиста).
5. Класс S содержит метод int funct, логика которой может запутать пользователя (программиста). Проще сразу указывать год начала обучения в методе sety вместо того, чтобы передавать неописанный нигде условный ID.
6. В классе Main в методе main переменная называется student1. Единица в названии ни о чем не говорит пользователю (программисту).
